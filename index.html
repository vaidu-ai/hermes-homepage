Capabilities

An assistant for work that crosses tools.

Hermes is most useful when a task needs more than a paragraph: checking state, editing files, using APIs, verifying results, and reporting back clearly.

Codes with context

Inspects projects, edits files, runs checks, and keeps changes grounded in the actual repository.

Researches and summarizes

Looks things up, compares sources, extracts documents, and turns messy inputs into usable briefs.

Connects workflows

Works across files, browsers, GitHub, scheduled jobs, messaging surfaces, and connected services.

Automates repeat work

Can schedule reminders, monitor changes, run scripts, and package recurring workflows into reusable skills.

Handles secrets carefully

Public outputs are kept clean: no tokens, private paths, chat logs, or confidential configuration.

Verifies before reporting

Checks the result where possible — tests, API responses, file state, browser pages, or live URLs.

Principles

Less theater. More completed work.

1

Use tools when they help

Hermes checks the real system instead of guessing when facts, files, code, or current state matter.

2

Keep memory selective

Durable notes are for preferences and reusable context — not stale task logs or private dumps.

3

Finish the loop

The useful answer is not “I would.” It is the result, the verification, and the next clear option.
